Abstract

This work proposes a novel Prime Bit Vedic multiplier architecture designed to enhance the efficiency of double-precision floating-point multiplication. Specifically targeting the mantissa multiplication stage, the methodology utilizes hierarchical Vedic blocks coupled with Prime Bit decomposition. The proposed approach aims to significantly reduce propagation delay and hardware utilization compared to traditional large-block multipliers. This document outlines the publication-aligned algorithm flow and provides access to the validated RTL implementation.

1. Introduction & Research Problem

The demand for high-speed double-precision floating-point arithmetic is ever-increasing in digital signal processing (DSP) and scientific computing workloads. However, the mantissa multiplication stage, involving 53-bit operands, often becomes a critical bottleneck in the hardware datapath. Straightforward implementations lead to significant delays and large area footprints.

2. Proposed Methodology

The core innovation lies in applying Prime Bit decomposition within a hierarchical Vedic multiplication framework. By decomposing the large multiplication into smaller, parallelizable Vedic blocks, partial products can be combined more efficiently.

2.1 Algorithm Flow

  1. Decomposition: Extract sign, exponent, and mantissa from 64-bit IEEE-754 inputs.
  2. Sign Computation: Result sign is generated via XOR of input sign bits.
  3. Exponent Addition: Sum exponents and apply bias correction (-1023).
  4. Mantissa Multiplication: Parallel 53x53 multiplication using hierarchical Vedic blocks.
  5. Post-Processing: Normalize the 106-bit product and adjust the exponent accordingly.
  6. Packing: Reconstruct the final IEEE-754 double-precision output.

3. RTL Implementation & Scope

The current hardware implementation in the repository provides a fully modular Verilog datapath. It focuses on functional correctness for normalized IEEE-754 finite test vectors.

3.1 Functional Scope

  • 64-bit floating-point datapath.
  • 53x53 Mantissa hierarchy.
  • Integrated normalization logic.

3.2 Limitations

  • NaN and Infinity handling (in progress).
  • Subnormal number handling.
  • Advanced rounding modes.

4. Reproducibility

Researchers can reproduce the results using Icarus Verilog or Vivado. The verification environment uses directed vectors from the IEEE-754 standard.
