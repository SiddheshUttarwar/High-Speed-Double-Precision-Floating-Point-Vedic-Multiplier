Publication-First Technical Explainer

Efficient Prime Bit Vedic Multiplier for Double Precision Floating Point Numbers

This page follows the published patent-journal record and explains the approach in plain language before showing how to reproduce the RTL implementation.

Published Document Repository

Abstract (Plain Language)

The publication proposes a Prime Bit Vedic multiplier method to improve double-precision floating-point multiplication efficiency, especially in the mantissa multiplication stage, with the goal of reducing delay and hardware cost.

1) Research Problem

Double-precision multipliers are essential in DSP and scientific workloads, but mantissa multiplication can become a major speed bottleneck in hardware implementations.

2) Proposed Idea

Apply Prime Bit decomposition with Vedic multiplication hierarchy so partial products can be combined efficiently and with lower delay impact than straightforward large-block multiplication.

3) Algorithm Flow (Publication-Aligned)

  1. Extract sign, exponent, and mantissa from two 64-bit inputs.
  2. Compute result sign with XOR.
  3. Add exponents and apply IEEE-754 bias correction.
  4. Multiply 53-bit mantissas using hierarchical Vedic blocks.
  5. Normalize product mantissa and adjust exponent if required.
  6. Pack final sign, exponent, and mantissa into output format.

6) What the Current RTL Implements

  • 64-bit datapath decomposition.
  • 53x53 mantissa multiplier hierarchy.
  • Exponent and normalizer blocks.
  • Icarus + Vivado runnable simulation flow.

7) Scope Limits

  • Focus on normalized finite test vectors.
  • Not a complete IEEE-754 corner-case engine yet.
  • NaN/Inf/subnormal handling needs extension for full compliance.
