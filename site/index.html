  • Core claim: Prime-bit Vedic multiplication can reduce delay/area impact for mantissa multiplication in double-precision arithmetic.

Problem Being Solved

Floating-point multiplication is used heavily in DSP, scientific computing, and accelerator pipelines. The slow part is often mantissa multiplication, which can become a timing bottleneck.

Plain-Language Idea

Instead of one large multiplier block, the method decomposes multiplication into smaller Vedic-style blocks and combines them efficiently. This can improve speed while keeping hardware practical.

Step-by-Step Algorithm (Simplified)

  1. Split each 64-bit input into sign, exponent, and mantissa fields.
  2. Compute output sign using XOR of input signs.
  3. Add exponents and subtract the IEEE-754 bias.
  4. Multiply 53-bit mantissas using hierarchical Vedic blocks.
  5. Normalize mantissa if product MSB requires shifting.
  6. Pack sign, exponent, and mantissa back into 64-bit output.

What the Current RTL Implements

  • Double-precision datapath decomposition.
  • 53x53 hierarchical Vedic mantissa multiplier.
  • Normalizer and exponent path logic.
  • User-ready simulation flows for Icarus and Vivado.

Current Limits (Important)

  • Focused on normalized finite operands.
  • NaN/Inf/subnormal edge behavior is not fully covered.
  • Not a complete IEEE-754 compliance core yet.
